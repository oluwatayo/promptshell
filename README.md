# promptshell

Turn a natural-language task into a shell script — and run it — from your terminal, powered by an LLM.

```sh
promptshell "find the 5 largest files in this directory and print their sizes"
```

promptshell sends your request to a language model, asks it to produce a shell
script for the task, writes that script to `prompt.sh`, makes it executable, and
runs it.

> [!WARNING]
> promptshell executes code generated by an LLM. Review what it produces before
> trusting it on anything important. A preview-and-confirm step is on the
> [roadmap](#roadmap); until then, treat the generated `prompt.sh` as untrusted.

## Status

Early stage. Today promptshell supports a single provider (Google Gemini) and a
minimal command-line interface. The [roadmap](#roadmap) below tracks the path to
a published, installable CLI with support for multiple LLM providers.

## Requirements

- Go 1.20+
- `bash` available on your `PATH`
- An API key for a supported provider (currently Google Gemini)

## Installation

> Installation via `go install` and prebuilt binaries is planned (see the
> [roadmap](#roadmap)). For now, build from source:

```sh
git clone https://github.com/oluwatayo/promptshell.git
cd promptshell
go build -o promptshell .
```

## Usage

```sh
./promptshell "<your task in plain English>"
```

Example:

```sh
./promptshell "compress all .log files in this folder into logs.tar.gz"
```

## Configuration

promptshell reads configuration from `~/.promptshell/config/config.json`:

```json
{
  "apiKey": "your-api-key",
  "provider": "gemini"
}
```

A multi-provider configuration format (with per-provider keys and model
selection) and environment-variable overrides are planned — see the roadmap.

## Roadmap

The goal is to grow promptshell from a single-file script into a published,
installable CLI that works with any of several LLM providers. Decisions locked
in: **CLI-first** (clean internals, not a stability-guaranteed library yet),
**MIT licensed**, with first-class support for **Gemini, OpenAI, Anthropic, and
Ollama (local)**.

### Target layout (CLI-first)

```
promptshell/
├── cmd/promptshell/main.go    # thin entrypoint
├── internal/cli/              # flags, subcommands (run, config)
├── internal/llm/              # Provider interface + registry
│   └── gemini/ openai/ anthropic/ ollama/
├── internal/config/           # v2 schema + migration from current format
├── internal/shell/            # extract → preview → confirm → execute
├── .github/workflows/         # ci.yml, release.yml
├── .goreleaser.yaml
└── LICENSE  README.md  CHANGELOG.md
```

### Phases

- **Phase 0 — Foundations**
  - Rename the module path to `github.com/oluwatayo/promptshell` (required for
    `go install` and importing).
  - Restructure into `cmd/` + `internal/`.
  - Add an MIT `LICENSE`; replace deprecated `io/ioutil`.
  - Rotate the previously committed API key.
- **Phase 1 — Provider interface**
  - Introduce a `Provider` interface (`Generate(ctx, Request) (Response, error)`)
    and a name-keyed registry.
  - Move the existing Gemini call behind the interface (no behavior change).
- **Phase 2 — Multi-LLM support**
  - Add OpenAI, Anthropic, and Ollama (local, no API key) providers.
  - Config v2: per-provider keys/models, `default_provider`, automatic migration
    from the current `{apiKey, provider}` format.
  - `--provider` / `--model` flags and `PROMPTSHELL_*` environment overrides.
- **Phase 3 — Safety & UX**
  - Preview the generated script and confirm before executing; add `--dry-run`
    and `--yes`.
  - Configurable shell (not hardcoded `bash`); robust script extraction;
    `--verbose` and cleaner error handling.
- **Phase 4 — Quality**
  - Unit tests with a mock provider, table tests for script extraction,
    `golangci-lint` and `go vet` in CI.
- **Phase 5 — Release & distribution**
  - SemVer tags and `CHANGELOG.md`.
  - GoReleaser + GitHub Actions to publish cross-platform binaries on tag.
  - Install via `go install github.com/oluwatayo/promptshell/cmd/promptshell@latest`,
    a Homebrew tap, or downloaded binaries.
- **Phase 6 — Stretch**
  - Streaming output, retries/backoff, conversation context, additional
    providers (Azure/Bedrock), custom system prompts, and an optional promoted
    public library API.

### Provider selection (planned)

Precedence: `--provider` flag → `PROMPTSHELL_PROVIDER` env var → `default_provider`
in config. The same order applies to model and API key. Providers that run
locally (Ollama) use a `base_url` instead of an API key.
